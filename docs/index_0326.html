1
/btw

2025 年 3 月 11 日上线 · Thariq 推文 200 万+ 阅读

不污染上下文的提问

Claude 正在重构一个大模块，你突然想问"测试文件在哪个目录"。以前只有两个选择：

直接问

Claude 停下来回答，上下文窗口多了一段不相关对话，重新干活可能跑偏。

等任务跑完再问

问完还是污染了上下文。而且你得傻等。

现在：/btw

插入一个问题，Claude 在并行进程中回答。回答完按空格或回车消除，对话历史干干净净。

# 使用方式

/btw 这个项目的抓取流程是什么？

# 回答完毕后

# 按 空格 或 回车 → 消除这段对话

# 之前的任务继续执行，零影响

几乎不费 token，因为复用了当前的提示缓存。属于用了就回不去的那种。

/rewind

快捷键：按两下 Esc · 2025 年 2 月升级分离回退

Claude Code 的 Ctrl+Z

以前改坏了只能整段对话一起回退，连讨论也没了。现在 /rewind 弹出菜单，让你精确选择：

回退代码和对话

完整撤销，回到选中时间点

只回退对话，保留代码

代码改对了，对话跑偏了

只回退代码，保留对话

讨论有用，代码重来

压缩对话释放上下文

窗口快满了，释放空间

最佳场景：做实验

让 Claude 试一种新方案 → 不行 → 代码回退、对话留着 → Claude 知道这条路不通 → 直接换方向，不用重新解释需求。

再也不用含泪 git reset 了。

/model opusplan

隐藏命令 · /model 列表里看不到

$20 Pro 用户的福音

自动在规划阶段使用 Opus，执行阶段切换到 Sonnet。

Opus 4.6 规划

深度思考 · 理解架构 · 分析依赖

Sonnet 4.6 执行

快速编码 · 完全够用 · 省额度

Pro 用户 Opus 额度有限。全程用 Opus 写代码，干到一半就限速。但规划和写代码对模型能力的要求完全不一样 — Opus 想清楚怎么做，Sonnet 去执行，两全其美。

# 直接输入（/model 列表里没有这个选项）

/simplify

2025 年 2 月底集成 · Boris 每天在用

找三个同事帮你同时 Review

输入后同时启动三个平行 Agent，从三个角度审查你的改动：

代码复用

是否有重复逻辑
能否抽取复用

代码质量

可读性、命名
边界情况处理

运行效率

性能瓶颈
更简洁的写法

AI 写的代码经常有微妙的冗余 — 多余的 import、重复逻辑、可以用更简洁写法替代的地方。/simplify 基本都能挑出来。

建议每次写完几个大功能后顺手跑一遍。比以前的 /review 好用得多。

10
快捷键

大部分人不知道的基本操作

四个必须知道的快捷键

Ctrl + V
直接粘贴截图

Debug 遇到报错？直接截屏粘过去，Claude 看图说话。不需要先保存文件再拖进去。

Mac 用户注意：这里是 Ctrl+V，不是 Cmd+V

Ctrl + J
换行

在输入框里换行，不会触发发送。Mac 用户也可以用 Option + Enter。

Ctrl + R
搜索 Prompt 历史

搜索你之前输入过的所有 prompt。上次那个好用的提示词，再也不用重新想了。

Ctrl + U
删除整行输入

写了一大段发现不对？一键清空当前输入，重来。
